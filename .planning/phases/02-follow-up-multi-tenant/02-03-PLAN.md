<objective>
Create two thin entry workflows — one per client. Each entry workflow contains exactly three functional nodes: (1) Webhook node with the client-specific URL slug, (2) Code node that validates the google_key and assembles the client_config object from non-sensitive config values + $env.* Twilio credentials, (3) Execute Sub-workflow node that calls the Core Workflow fire-and-forget (waitForSubWorkflow: false).

Purpose: Each client gets a unique webhook URL (CONF-03). Adding a third client requires only creating another entry workflow — no changes to the Core Workflow (CONF-02). Credential isolation is enforced by construction: each entry workflow reads only its own CLIENTSLUG_* env vars.

Output: Two importable n8n workflow JSON files, one per client.
</objective>

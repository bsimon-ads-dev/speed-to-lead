WhatsApp HTTP Request node format (verified in 03-RESEARCH.md Pattern 2):
  method: POST
  url: =https://api.twilio.com/2010-04-01/Accounts/{{ $json.client_config.twilio_account_sid }}/Messages.json
  Authorization: =Basic {{ Buffer.from($json.client_config.twilio_account_sid + ':' + $json.client_config.twilio_auth_token).toString('base64') }}
  specifyBody: keypair
  bodyParameters:
    To:               =whatsapp:{{ $json.phone }}              (prospect) / =whatsapp:{{ $json.client_config.owner_phone }} (owner)
    From:             =whatsapp:{{ $json.client_config.whatsapp_sender }}
    ContentSid:       ={{ $json.client_config.whatsapp_template_sid }}   (prospect) / ={{ $json.client_config.whatsapp_owner_template_sid }} (owner)
    ContentVariables: ={"1":"{{ $json.name }}","2":"{{ $json.client_config.business_name }}","3":"{{ $json.client_config.callback_promise_minutes }}"}  (prospect)
                      ={"1":"{{ $json.name }}","2":"{{ $json.request }}","3":"{{ $json.phone }}"}  (owner)

CRITICAL: Do NOT add a Body parameter to WhatsApp nodes. Body on a WhatsApp send is ignored for new contacts and can cause silent delivery failure or WABA flagging.
CRITICAL: whatsapp: prefix is hardcoded in the expression (=whatsapp:{{ $json.phone }}), NOT stored in client_config — prevents prefix being accidentally omitted.

ContentVariables format: JSON string literal in the n8n expression. Do NOT use JSON.stringify() in the expression field — n8n expression engine double-encodes it. The expression ={"1":"{{ $json.name }}"} already evaluates to a JSON string.

Entry workflow client_config assembler pattern (from Code: Assemble Client Config in entry-dupont-plomberie.json):
  twilio_account_sid: $env.DUPONT_TWILIO_ACCOUNT_SID,
  twilio_auth_token: $env.DUPONT_TWILIO_AUTH_TOKEN,
  New fields follow same pattern:
  whatsapp_sender: $env.DUPONT_WHATSAPP_SENDER,
  whatsapp_template_sid: $env.DUPONT_WHATSAPP_TEMPLATE_SID,
  whatsapp_owner_template_sid: $env.DUPONT_WHATSAPP_OWNER_TEMPLATE_SID,
  whatsapp_enabled: false,         // hardcoded false until client completes onboarding
  owner_whatsapp_enabled: false
